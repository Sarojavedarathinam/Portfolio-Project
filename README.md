## 🚧 Issues I Ran Into — And How I Solved Them

The real learning happened here. Every error made me understand the tools better.

❌ Issue 1 — Opened the wrong app
I accidentally downloaded a different tool that showed an Automations
and Bugbot screen instead of a code editor.
✅ Fixed: Went directly to cursor.com and downloaded the correct
Cursor IDE from the official website.

❌ Issue 2 — Typed the $ sign in the terminal
I copied the $ sign from the guide into the terminal which gave me
a command not found error.
✅ Fixed: Learned that $ is just a guide symbol and should never
be typed. Commands always start directly with the word git.

❌ Issue 3 — Typed "YourName" literally
I used "YourName" in the git clone link instead of my real GitHub
username and got a repository not found error.
✅ Fixed: Found my real username on github.com (top right corner)
and used Sarojavedarathinam in the command. It worked immediately.

❌ Issue 4 — Git was not installed
Running git clone gave the error "git is not recognized as a command."
✅ Fixed: Downloaded and installed Git from git-scm.com, restarted
Cursor and ran the command again successfully.

❌ Issue 5 — Author identity unknown
Git commit failed saying it could not detect my email address.
✅ Fixed: Ran git config --global user.email and git config
--global user.name to register my identity with Git. Commit
worked after that.

❌ Issue 6 — Push was rejected
Git push failed saying the remote contains work I do not have locally.
✅ Fixed: Ran git pull --rebase to sync the latest version from
GitHub first, then ran git push again. README uploaded successfully.
